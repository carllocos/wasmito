### Extending the debugger to target a language

By default every language that compiles to WebAssembly can be targetted by the debugger as long as `DWARF` or the `Source Map Spec` is generated by the compiler.
By default the following debug operations are available for all the languages:

- `step into`
- `step over`
- `step out`
- `step to next iteration`
- `add breakpoint`
- `remove breakpoint`

If you desire to make other debug operations available to your language.
See next section.

### Configure Debugger to support more debug operations on your language

To extend your language with additional debugging operations.
Additional requirements need to be met.

1. Have an AST `tree-sitter` parser available for your language
2. Compile the `tree-sitter` to WebAssembly. You can find a detailed explaination on the official website.
3. Add an entry to the `languages.json` for your language (see below)

For 3, follow the json configuration format:

```JSON
{
    "language": "name-of-your-language",
    "fileExtensions": ["ext1", "ext2"],
    "parserName": "tree-sitter-parser.wasm",
    "astDebugOperations":[]
}
```

The `astDebugOperations` expects an array of objects of the following format:

```JSON
{
    "astNodeDescription": {
        "grammarID": number-of-node-id,
        "grammarType": "name-of-type"
    },
    "debugOperation": "name-of-operation"
}
```

The following `debugOperations` are supported:

- `"BreakOnInstanceCreation"`: break program execution on every instrance creation
